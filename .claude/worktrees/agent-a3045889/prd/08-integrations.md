## Overview

Doable supports a rich ecosystem of integrations through three connector types: **Shared Connectors** (platform-wide, admin-configured), **Personal Connectors** (MCP servers for individual build-time context), and **Any API** (custom authenticated endpoints). This replaces the previous "Integrations" naming with "Connectors."

---

## 1. Connector Types

### 1.1 Shared Connectors
| Feature | Description |
|---------|-------------|
| **Scope** | Workspace-wide, configured once by admin |
| **Access** | Available to all workspace members |
| **Purpose** | Add functionality to built apps |
| **Examples** | AI, Stripe, Shopify, Firecrawl, Perplexity, Slack, ElevenLabs |

### 1.2 Personal Connectors (MCP Servers)
| Feature | Description |
|---------|-------------|
| **Scope** | Individual user only |
| **Access** | Only accessible to the user who configured them |
| **Purpose** | Build-time context from personal tools |
| **Examples** | Linear, Notion, Miro, custom tools |
| **Protocol** | Model Context Protocol (MCP) |

### 1.3 Any API (Authenticated)
| Feature | Description |
|---------|-------------|
| **Scope** | Per-project |
| **Access** | Available in edge functions |
| **Auth methods** | API key, Bearer token, OAuth |
| **Configuration** | Prompt with endpoint details, auth method, headers, OpenAPI spec |
| **Credentials** | Stored as encrypted secrets, injected into edge functions |
| **Security** | Never exposed in frontend code |

### 1.4 Gateway-Based Connectors
Some shared connectors use a **secure gateway** between the Doable app and the third-party service:

| Feature | Description |
|---------|-------------|
| **Auto OAuth** | Gateway manages OAuth tokens and refreshes expiring tokens automatically |
| **No re-auth** | Once connected, integration keeps working without manual re-authentication |
| **Credentials hidden** | Tokens stored securely in gateway, not visible in project settings or to admins |
| **Transparent** | App calls the connector; auth is handled behind the scenes |
| **Examples** | Slack (gateway-based) |

### 1.5 Connector Admin Controls (Business/Enterprise)
| Feature | Description |
|---------|-------------|
| **Enable/Disable shared connectors** | Admins toggle specific shared connectors via Settings → Connectors → Shared connectors |
| **MCP access toggle** | Settings → Privacy & security → Enable/disable MCP access |
| **Per-connector control** | Cogwheel icon on each connector for settings |
| **Doable Cloud** | Cannot be disabled at workspace level |
| **Custom MCP servers** | Business/Enterprise can connect custom MCP servers as personal connectors |

## 3. Stripe Payment Integration

### 3.1 Setup Flow
1. Connect Doable Cloud (database must be connected first)
2. Add Stripe Secret Key via secure "Add API Key" form
3. **Never paste keys in chat** — always use the secure form
4. Prompt for payment features (e.g., "Add $29 one-time checkout" or "Three subscription tiers")

### 3.2 Auto-Generated Components
| Component | Description |
|-----------|-------------|
| **Edge Functions** | Checkout session creation, customer portal, subscription polling |
| **Database tables** | Users, subscriptions, payments with RLS |
| **UI components** | Pricing cards, checkout buttons, subscription management |
| **Webhook handler** | Stripe event processing (opt-in) |
| **User linking** | Payments tied to authenticated user `id` |

### 3.3 Payment Types
| Type | Description |
|------|-------------|
| **One-time payments** | Single charge checkouts |
| **Subscriptions** | Recurring billing with tiers |
| **Customer portal** | Self-service billing management |
| **Webhooks** | Real-time payment event handling |

### 3.4 Testing
- Use Stripe test mode
- Test card: `4242 4242 4242 4242`
- **Requires deployed app** (not preview mode)

### 4.1 Import Flow
1. Install Builder.io Figma plugin
2. Select frames in Figma

4. Open in Doable for AI refinement

### 7.3 Personal Connector Workflow
- Connect via Settings → Connectors → Personal connectors
- Sign in and grant access to personal account
- Only the connecting user can access
- Not included in deployed app (build-time only)
- Example prompts:
  - "Generate a prototype based on tickets in the current sprint" (Linear/Jira)
  - "Turn our Q4 campaign brief into a landing page" (Notion/Confluence)
  - "Prototype the signup flow from our whiteboard" (Miro)

### 7.2 Webhook Pattern
```
External Service → Webhook → Edge Function → Database → Frontend Update
```
