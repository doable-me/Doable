# BUG-FOLDER-002: parentId from different workspace causes 500

**Date**: 2026-05-14
**Severity**: High
**Feature**: Folders — cross-workspace parentId validation
**Environment**: dev (https://dev-api.doable.me)

## Steps to Reproduce

1. Authenticate as owner-pro@doable.me (workspace A)
2. Obtain a folder ID from workspace B (another workspace where user is also a member)
3. POST /folders with body `{"name": "test", "workspaceId": "{workspaceA}", "parentId": "{folderFromWorkspaceB}"}`

## Expected

400 Bad Request — parentId must belong to the same workspace

## Actual

500 Internal Server Error — no cross-workspace check on parentId; DB constraint or FK violation causes unhandled exception

## Impact

Server crashes instead of returning a validation error. Could also allow cross-workspace folder tree pollution if the DB constraint is not enforced.
