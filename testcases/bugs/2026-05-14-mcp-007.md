# BUG-2026-05-14-MCP-007 — HTTP connector accepts non-https URLs (security issue)

**Date:** 2026-05-14
**Env:** dev (dev-api.doable.me)
**Severity:** HIGH
**Status:** OPEN
**Test Case:** TC-MCP-CONNECTOR-009

## Description

The connector API accepts http:// (non-HTTPS) URLs when creating HTTP-type MCP connectors. This should be rejected to prevent plaintext transmission of credentials and tool invocations.

## Evidence

```
POST /workspaces/:wsId/connectors
{"name":"insecure-http","transportType":"streamable_http","scope":"workspace","serverUrl":"http://example.com/mcp"}
Response: HTTP 201 - connector created successfully with server_url="http://example.com/mcp"
```

## Expected

HTTP 400 with error indicating HTTPS is required for remote MCP connectors.

## Impact

Users can configure MCP connectors pointing to plain HTTP endpoints, exposing tool arguments and results in plaintext.
