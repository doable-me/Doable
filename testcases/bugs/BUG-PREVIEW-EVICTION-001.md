# BUG-PREVIEW-EVICTION-001 — Active dev-server evicted from registry under concurrent load

**Severity:** medium (UX — preview goes 503 mid-session)
**Found:** 2026-05-10 by lead during parallel-app-matrix session on env1
**Test:** Live observation; not yet a formal TC

## Reproduction

1. Spawn 6+ concurrent vite dev-servers via parallel chat sessions (each agent created a fresh project ⇒ a fresh dev-server). Server `/admin/dev-servers` reports `total: 6, alive: 6, ready: 6` for several minutes with combined memory ~960 MB.
2. Continue interacting with the projects.
3. Open editor for one of the older projects (e.g. project `3f3dd37c-276a-426f-8632-d4c89b0eb6a0` — multipage app, age ~10 min) — the editor pane shows `Preparing live preview… / Creating project files…` indefinitely.

## Actual

- `GET /admin/dev-servers` no longer lists project `3f3dd37c…`. The dev-server has been **removed from the in-memory registry**.
- `GET /preview/3f3dd37c…/` returns **HTTP 503**.
- The project's source files on disk are intact (App.tsx + Navigation.tsx + 12 files generated by turn 1).
- No SSE event was sent to clients of that project announcing the eviction.

## Expected

When the dev-server pool needs to evict (memory pressure, max-N policy, idle timeout), the editor should:
- Show a "Dev server is asleep — wake up?" CTA in the preview pane (instead of an indefinite spinner), and
- Re-spawn vite on user interaction (page open, next chat turn, or explicit "Wake preview" button).

The eviction itself is reasonable for resource control — silent eviction with a stuck UI is the bug.

## Suggested fix

- In `services/api/src/runtime/` find the dev-server LRU/idle-eviction path. After eviction, mark the project as `ready: false, alive: false, hibernated: true` and surface that state in `/admin/dev-servers` AND in the per-project `dev_server_status` so the editor can render a Wake CTA.
- In `apps/web/src/app/editor/[projectId]/page.tsx`, when the dev-server status is `hibernated`, replace the indefinite spinner with a "Wake preview" button that POSTs `/projects/:id/dev-server/wake` (or simply trigger a follow-up no-op chat to provoke spawn).
