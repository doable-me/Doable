# BUG-EDITOR-001: POST /projects/:id/files returns 201 but file is not retrievable via GET

**Date:** 2026-05-14
**Severity:** HIGH
**Environment:** dev (dev-api.doable.me)
**Test Case:** TC-EDITOR-FILES-005, TC-EDITOR-FILES-053

## Summary

`POST /projects/:id/files` returns HTTP 201 and reports the file was created, but the file does NOT appear in the GET `/projects/:id/files` tree response, and `GET /projects/:id/files/:path` returns 404 for the same path.

## Repro Steps

```
TOK=<valid-jwt>
PROJ=a9bcb1a9-20ea-4ad5-a4e3-1ed9662284ac

# 1. Create file via POST
curl -X POST https://dev-api.doable.me/projects/$PROJ/files \
  -H "Authorization: Bearer $TOK" \
  -H "Content-Type: application/json" \
  -d '{"path":"probe.txt","content":"hello"}'
# Response: 201 {"data":{"path":"probe.txt","updatedAt":"..."}}

# 2. Read it back via GET
curl https://dev-api.doable.me/projects/$PROJ/files/probe.txt \
  -H "Authorization: Bearer $TOK"
# Response: 404 {"error":"File not found: probe.txt"}

# 3. Check tree
curl https://dev-api.doable.me/projects/$PROJ/files \
  -H "Authorization: Bearer $TOK"
# Response: file NOT present in tree
```

## Expected

File created via POST should be immediately readable via GET `/projects/:id/files/:path` and visible in the GET `/projects/:id/files` tree.

## Actual

- POST returns 201 (success)
- GET by path returns 404 "File not found"
- File is absent from the tree

## Root Cause (hypothesis)

`POST /projects/:id/files` likely writes to the **DB-backed `project_files` table** while `GET /projects/:id/files` and `GET /projects/:id/files/:path` read from the **in-memory store** (`editor.ts` Map). These are two separate backends that are not synchronized. Files written via POST are invisible to the in-memory GET routes.

Note: `PUT /projects/:id/files/:path` (upsert) correctly writes to the in-memory store and IS readable by GET. Only the POST endpoint is broken.

## Impact

Any workflow that creates a file via POST (e.g. "New File" dialog in the editor) results in a silent data loss — the file appears created but cannot be read or edited.
