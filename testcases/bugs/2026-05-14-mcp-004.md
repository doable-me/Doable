# BUG-2026-05-14-MCP-004 — Integration catalog returns 404 at /integrations (wrong route)

**Date:** 2026-05-14
**Env:** dev (dev-api.doable.me)
**Severity:** MEDIUM
**Status:** OPEN
**Test Case:** TC-INTEG-LIST-001

## Description

The integration listing endpoint at GET /integrations returns HTTP 404. The catalog is actually at /integrations/catalog instead.

## Evidence

```
GET /integrations → HTTP 404 {"error":"Not Found","path":"/integrations"}
GET /integrations/catalog → HTTP 200 (returns 533 integrations)
```

## Expected

GET /integrations should return the catalog (per TC-INTEG-LIST-001 spec), or redirect to /integrations/catalog.

## Note

The test spec says the route is GET /integrations but the actual API route is GET /integrations/catalog. This is a route naming discrepancy.
