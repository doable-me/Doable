# BUG-025: GET /ai/models returns "Not authenticated" for all users

**Date:** 2026-05-14
**Suite:** 05-ai-chat
**Severity:** High
**Test Cases:** TC-AI-CHAT-MODELS-001, TC-AI-CHAT-MODELS-002

## Description
`GET /ai/models` returns an error for all authenticated users including platform admins. The endpoint exists and returns HTTP 200 but with an empty data array and an error message indicating Copilot CLI authentication failure.

## Reproduction
```
GET https://dev-api.doable.me/ai/models
Authorization: Bearer <any-valid-token>
```

## Actual Response
```json
{"data":[],"error":"Request models.list failed with message: Not authenticated. Please authenticate first."}
```

## Expected
```json
{"data":[{"id":"copilot/gpt-4.1","label":"GPT-4.1","provider":"copilot","enabled":true},...]}
```

## Root Cause (suspected)
The dev server's GitHub Copilot CLI is not authenticated. The `models.list` request to the Copilot CLI backend fails because there is no active Copilot session on the dev server.

## Impact
Model discovery is broken. Clients cannot list available models. Model switching UI cannot populate. All TC-AI-CHAT-MODELS tests requiring model list are blocked.
