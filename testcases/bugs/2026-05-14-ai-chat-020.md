# BUG-020: Zero monthly credit balance not enforced — sends succeed when monthlyRemaining=0

**Date:** 2026-05-14
**Suite:** 05-ai-chat
**Severity:** Critical
**Test Cases:** TC-AI-CHAT-CREDITS-009, TC-AI-CHAT-CREDITS-014

## Description
A free workspace with `monthlyRemaining: 0` and `monthlyMax: 0` still successfully processes AI chat requests. The `requireCredits` middleware is not blocking sends when the monthly quota is exhausted.

## Evidence
```
GET /billing/balance?workspaceId=0b302361-4a50-4c9b-a985-49dcdf8eab23
=> {"dailyRemaining":4,"dailyMax":5,"monthlyRemaining":0,"monthlyMax":0,...,"planType":"free"}

POST /projects/6e09bbd1.../chat {"content":"hello","mode":"agent"}
=> HTTP 200 + SSE stream begins (scaffolding, installing deps...)
```

## Expected
HTTP 429 with body `{"error":"insufficient_credits","retryAfter":<seconds>}` when monthlyRemaining=0.

## Impact
Free-tier users can exceed their monthly credit quota indefinitely. Revenue impact: compute costs incurred without billing.
