# BUG-API-013: GET /integrations/connections requires undocumented workspaceId param

**Date:** 2026-05-14
**Severity:** medium
**Area:** TC-API-INTEG-006

## Summary
`GET /integrations/connections` returns 400 with `{"error":"workspaceId query parameter is required"}` even for authenticated users. The test spec expects 200 with a list of user's connections. The required `workspaceId` is not documented in the test spec.

## Steps to Reproduce
```bash
curl "https://dev-api.doable.me/integrations/connections" \
  -H "Authorization: Bearer $TOKEN"
# HTTP 400: {"error":"workspaceId query parameter is required"}

curl "https://dev-api.doable.me/integrations/connections?workspaceId=<wid>" \
  -H "Authorization: Bearer $TOKEN"
# HTTP 200: {"data":[]}
```

## Expected
200 with user's connections (workspaceId optional or inferred).

## Impact
All clients must pass workspaceId explicitly. Any client built to the spec without workspaceId will get 400.
