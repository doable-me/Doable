# BUG-2026-05-14-MCP-003 — /integrations/enhanced-auth/start route returns 404

**Date:** 2026-05-14
**Env:** dev (dev-api.doable.me)  
**Severity:** HIGH
**Status:** OPEN
**Test Case:** TC-INTEG-CONNECT-001

## Description

The integration OAuth start endpoint `/integrations/enhanced-auth/start?slug=github` returns HTTP 404 "Not Found". The route does not exist on the dev API.

## Evidence

```
GET https://dev-api.doable.me/integrations/enhanced-auth/start?slug=github
Response: HTTP 404 {"error":"Not Found","path":"/integrations/enhanced-auth/start"}
```

Also tested:
- GET /integrations/oauth/github/authorize → HTTP 400 (route exists but has different behavior)
- GET /integrations/enhanced-auth/github/start → HTTP 404

## Expected

HTTP 302 redirect to GitHub OAuth with state + code_challenge (PKCE).

## Impact

OAuth-based integration connection flow is completely broken. Users cannot connect any OAuth integrations.
