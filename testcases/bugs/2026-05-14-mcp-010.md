# BUG-2026-05-14-MCP-010 — Connector disable (enabled:false) doesn't change status

**Date:** 2026-05-14
**Env:** dev (dev-api.doable.me)
**Severity:** HIGH
**Status:** OPEN
**Test Case:** TC-MCP-CONNECTOR-014

## Description

PATCH /workspaces/:wsId/connectors/:id with {"enabled":false} returns HTTP 200 but the connector status remains "active". The `enabled` field is not reflected in the response and the connector is not actually disabled.

## Evidence

```
PATCH /workspaces/492bdda5.../connectors/19021c67.../
Body: {"enabled":false}
Response: HTTP 200 with status="active" (unchanged)
```

There is no `enabled` field in the connector response schema - the concept of "enabled" may not be implemented.

## Expected

PATCH {enabled:false} should stop the connector process and set status to "inactive" or "disabled".

## Impact

Users cannot disable connectors without deleting them. Disabling is useful for temporary deactivation without losing configuration.
