# BUG-2026-05-14-publish-006: Dev API server crashed to 502 during test session

**Severity:** High  
**Environment:** dev (https://dev.doable.me / https://dev-api.doable.me)  
**Date:** 2026-05-14  
**Affected TCs:** All API-dependent test cases

## Summary
The dev API server (dev-api.doable.me) went from healthy (200) to 502 Bad Gateway during the test session at approximately 17:58 UTC. Cloudflare tunnel is healthy but the origin server stopped responding.

## Timeline
- 17:45 UTC: API healthy — `{"status":"healthy","uptime":8.8s}`
- 17:58 UTC: API returning 502 Bad Gateway from Cloudflare (Chennai edge)

## Evidence
Screenshot shows: You (Browser) — Working, Chennai (Cloudflare) — Working, dev-api.doable.me (Host) — Error

## Impact
All API-dependent testing blocked after 17:58 UTC. Cannot verify deploy endpoints, deployment history, custom domains, or runtime tests.

## Notes
The API may have crashed due to the deploy attempt triggering the filesystem error (ENOENT during mkdir). The error could have caused an unhandled exception that brought down the Node.js process. The `tsx watch` process should auto-restart but may not have if the crash was in the wrong location.
