# BUG-2026-05-14-WS-003: GET /projects/shared returns 502 Bad Gateway

**Date:** 2026-05-14
**Environment:** dev (dev-api.doable.me)
**Severity:** High
**Category:** Runtime Crash / 502
**Test Case:** TC-PROJ-LIST-049

## Summary
`GET /projects/shared` consistently returns `502 Bad Gateway` via Cloudflare, indicating the API process crashes or times out when handling this route.

## Steps to Reproduce
1. Authenticate as `owner-pro@doable.me`
2. `GET https://dev-api.doable.me/projects/shared`
3. Observe: `HTTP 502 Bad Gateway` (Cloudflare error, not application JSON)

## Expected
`200 {"data":[], "pagination":{...}}` (empty list if user has no shared projects)

## Actual
`502 Bad Gateway` — Cloudflare proxy receives no response or connection reset from API

## Additional Context
- Confirmed with `curl -sv` — CF-RAY header present, upstream error
- `/projects/starred` and `/projects/recently-viewed` both return `200` correctly
- Only `/projects/shared` is affected
- Occurred consistently across multiple requests with valid auth tokens

## Impact
- The "Shared with me" projects view is completely non-functional
- Any UI relying on this endpoint will fail to load

## Recommended Fix
Investigate the `/projects/shared` route handler in `services/api/src/routes/projects.ts`. Likely an unhandled async error, missing await, or broken DB query that crashes the process. Add error boundary / try-catch.
