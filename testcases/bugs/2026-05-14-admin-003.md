# BUG-ADMIN-003: /admin/projects sorting not applied — results always in same order

**Date:** 2026-05-14
**Severity:** High
**Test Cases:** TC-ADMIN-PROJECTS-009, TC-ADMIN-PROJECTS-010

## Summary
GET /admin/projects?sort=name&dir=asc and ?sort=name&dir=desc both return results in the same unsorted order. The `sort` and `dir` query parameters are silently ignored.

## Steps to Reproduce
```
GET https://dev-api.doable.me/admin/projects?sort=name&dir=asc
GET https://dev-api.doable.me/admin/projects?sort=name&dir=desc
```

## Expected
- dir=asc: first 5 projects sorted alphabetically ascending by name
- dir=desc: first 5 projects sorted alphabetically descending by name

## Actual
Both return same order: `['Updated Project Name', 'MCP QA Test Project', 'AI Chat Test', 'Smoke Vite Updated', 'Free Proj 3']`

Python sorted check: `Actually sorted asc: False` / `Actually sorted desc: False`

## Impact
Admin cannot sort project list by any column. Sort controls in the UI are non-functional.
