# BUG-2026-05-14-BILLING-006: Login rate-limiter shows generic "Something went wrong" instead of "Too many requests"

**Severity:** Medium
**Area:** Auth UI — Login form error messaging
**URL:** https://dev.doable.me/login
**Discovered:** 2026-05-14

## Description
When the login rate limiter triggers, the API returns `{"error":"Too many requests, please try again later."}`. However, the UI displays a generic "Something went wrong. Please try again." message instead of the specific rate-limit message.

## Steps to Reproduce
1. Submit login form several times in quick succession
2. After rate limit kicks in, observe the error banner

## Expected
- "Too many requests, please try again later." (or similar informative message)

## Actual
- "Something went wrong. Please try again." — unhelpful, user doesn't know to wait

## Impact
Poor UX — users may think their credentials are wrong and keep trying, worsening the rate limit situation.
