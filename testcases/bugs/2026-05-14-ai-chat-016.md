# BUG-016: Idempotency-Key header not honored — duplicate messages created

**Date:** 2026-05-14
**Suite:** 05-ai-chat
**Severity:** High
**Test Cases:** TC-AI-CHAT-SEND-044

## Description
Sending the same request with the same `Idempotency-Key` header twice creates two separate user messages in history. The server does not detect the duplicate and processes both requests independently.

## Reproduction
```
# Request 1
POST https://dev-api.doable.me/projects/:id/chat
Idempotency-Key: test-idem-key-001
{"content":"idempotency test","mode":"agent"}
# => HTTP 200 + SSE stream completes

# Request 2 (same key, same content)
POST https://dev-api.doable.me/projects/:id/chat
Idempotency-Key: test-idem-key-001
{"content":"idempotency test","mode":"agent"}
# => HTTP 200 + SSE stream completes (should return prior result)
```

## Actual
Both requests stream independently. History shows `"idempotency test"` appearing twice in user messages.

## Expected
Second request returns cached result; no duplicate message row; no duplicate credit deduction.

## Impact
Network retries or client bugs can cause duplicate AI turns and double credit charges.
