# BUG-VER-002: Restore with invalid SHA returns 500 instead of 404

**Date**: 2026-05-14
**Severity**: Medium
**Feature**: Versions — restore error handling
**Environment**: dev (https://dev-api.doable.me)

## Steps to Reproduce

1. Authenticate as owner-pro@doable.me
2. POST https://dev-api.doable.me/projects/{projectId}/versions/{invalidSha}/restore
   - Use a random 40-char hex string that does not correspond to any real git SHA

## Expected

404 Not Found — version/SHA does not exist

## Actual

500 Internal Server Error — git command fails on invalid SHA and the error propagates unhandled

## Impact

Clients cannot distinguish "version not found" from a server crash. Error handling is insufficient around git restore operations.
