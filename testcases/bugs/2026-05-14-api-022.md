# BUG-API-022: GET /thumbnails/:projectId returns 400 instead of 404/200

**Date:** 2026-05-14
**Severity:** medium
**Area:** TC-API-THUMB-001, TC-API-THUMB-003

## Summary
`GET /thumbnails/:projectId` returns 400 with `{"error":"Only .png thumbnails are supported"}` for a project that has no thumbnail yet. This is the wrong error type — a GET request on a resource that hasn't been generated should return 404 or a placeholder, not a validation error about file extension.

## Steps to Reproduce
```bash
curl "https://dev-api.doable.me/thumbnails/<project-with-no-thumbnail>" \
  -H "Authorization: Bearer $TOKEN"
# HTTP 400: {"error":"Only .png thumbnails are supported"}
```

## Expected
Either:
- 404 Not Found (thumbnail not generated yet)
- 200 with placeholder image

## Impact
Clients receive a misleading 400 error when checking for a thumbnail on a new project. The error message is about file format, not about "no thumbnail exists".
