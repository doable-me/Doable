# BUG-API-003: GET /projects/:id returns 400 for non-existent valid UUID

**Date:** 2026-05-14
**Severity:** high
**Area:** TC-API-PROJECTS-033

## Summary
`GET /projects/00000000-0000-0000-0000-000000000000` (a valid UUID that does not exist) returns 400 "Invalid project id" instead of 404 Not Found. This conflates "bad input" (400) with "not found" (404).

## Steps to Reproduce
```bash
curl https://dev-api.doable.me/projects/00000000-0000-0000-0000-000000000000 \
  -H "Authorization: Bearer $TOKEN"
```

## Actual Response
```
HTTP/1.1 400 Bad Request
{"error":"Invalid project id"}
```

## Expected
```
HTTP/1.1 404 Not Found
{"error":"Project not found"}
```

## Impact
Clients cannot distinguish between "you sent a malformed ID" (400) and "that project doesn't exist" (404). API contract broken per REST conventions.
