# BUG-COLLAB-002: dev-api.doable.me returns 502 Bad Gateway on all endpoints

**Date**: 2026-05-14
**Environment**: dev (dev-api.doable.me)
**Severity**: CRITICAL
**Area**: Infrastructure / API Server

## Summary
The dev API server (dev-api.doable.me) is returning HTTP 502 Bad Gateway on all endpoints, indicating the backend API service is down or the Cloudflare Tunnel / Caddy reverse proxy cannot reach the API process.

## Steps to Reproduce
1. Make any HTTP request to dev-api.doable.me
2. Observe 502 response

## Expected
API server should return 200 on /health and process requests normally.

## Actual
- `GET https://dev-api.doable.me/health` → 502 Bad Gateway
- `POST https://dev-api.doable.me/auth/login` → 502 Bad Gateway
- `GET https://dev-api.doable.me/projects` → 502 (observed via browser console)
- WebSocket `wss://dev-api.doable.me/preview/{id}/` → 502 (WebSocket handshake failure)

## Evidence
PowerShell verification:
```
FAIL [502] Response status code does not indicate success: 502 (Bad Gateway): https://dev-api.doable.me/health
FAIL [502] Response status code does not indicate success: 502 (Bad Gateway): https://dev-api.doable.me/auth/login
```
Browser console (186 errors):
- `WebSocket connection to 'wss://dev-api.doable.me/preview/f42fcc90-.../' failed: Unexpected response code: 502`
- `Failed to load resource: 502 () @ https://dev-api.doable.me/projects/.../runtime/metrics`

## Contrast
- dev.doable.me (web frontend): 200 OK
- dev-ws.doable.me (WS server): 200 OK
- dev-api.doable.me (API): 502 FAIL

## Impact
Complete loss of functionality in dev environment. No login, no project access, no AI chat, no collaboration features testable.
