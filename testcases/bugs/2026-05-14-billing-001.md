# BUG-2026-05-14-BILLING-001: Monthly Credits display shows wrong value (-400 / 100 used)

**Severity:** High
**Area:** Billing UI — Credits Panel
**URL:** https://dev.doable.me/billing
**User:** owner-pro@doable.me (Pro Plan)
**Discovered:** 2026-05-14

## Description
The Monthly Credits row on the Billing page shows "-400 / 100 used" with a full progress bar (100%), but the API returns `monthlyRemaining=500, monthlyMax=500` (no credits used).

## Steps to Reproduce
1. Log in as owner-pro@doable.me
2. Navigate to https://dev.doable.me/billing
3. Observe the "Monthly Credits" row

## Expected
- Monthly Credits: 0 / 500 used (or 500 remaining)
- Progress bar near empty

## Actual
- Monthly Credits: -400 / 100 used
- Progress bar fully filled (purple, 100%)
- "549 available" badge shown (incorrect calculation)

## API Evidence
```
GET /billing/balance?workspaceId=492bdda5-...
Response: {"dailyRemaining":49,"dailyMax":50,"monthlyRemaining":500,"monthlyMax":500,"topupRemaining":0}
```

## Root Cause (suspected)
UI formula bug: computing `monthlyMax - monthlyRemaining` as "used" but using wrong fields, possibly mixing up daily/monthly values.
