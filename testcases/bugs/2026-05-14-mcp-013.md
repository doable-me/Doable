# BUG-2026-05-14-MCP-013 — MCP Apps: mcp_ui_resource event not emitted on dev environment

**Date:** 2026-05-14
**Env:** dev (dev-api.doable.me)
**Severity:** HIGH
**Status:** OPEN
**Test Case:** TC-MCP-APPS-RUNTIME-001

## Description

When invoking the Markdown Builder MCP App via chat on the dev environment, the SSE stream includes `tool_call` and `tool_result` events but NO `mcp_ui_resource` event is emitted. The MCP App UI resource (iframe HTML) is not being returned.

## Evidence

Chat SSE event types observed:
```
thinking: 7, status: 5, tool_call: 2, thinking_block_end: 1, tool_result: 1
Has mcp_ui_resource: False
```

Expected (from TC-MCP-APPS-RUNTIME-001 which passed on zantaz/env1):
```
mcp_ui_resource: 1  (with resource.uri starting with "ui://")
```

## Comparison

On the zantaz environment (env1), the same test produced a `mcp_ui_resource` event with 8,605 bytes of iframe HTML. On dev, this event is absent.

## Possible Causes

1. The Markdown Builder MCP server binary at `/root/doable/mcp-servers/markdown-builder/index.mjs` may not be installed on the dev server.
2. The MCP App connector may not be spawned correctly on dev.
3. The Markdown Builder may be returning a plain text result instead of a `ui://` resource.

## Impact

MCP Apps (the core "Doable is MCP Apps Compatible" claim) are not functional on dev environment.
