# BUG-API-004: DELETE /projects/:id returns 200 instead of 204

**Date:** 2026-05-14
**Severity:** low
**Area:** TC-API-PROJECTS-043

## Summary
`DELETE /projects/:id` returns HTTP 200 instead of the expected 204 No Content.

## Steps to Reproduce
```bash
curl -si -X DELETE "https://dev-api.doable.me/projects/<pid>" \
  -H "Authorization: Bearer $TOKEN"
```

## Actual Response
```
HTTP/1.1 200 OK
```

## Expected
```
HTTP/1.1 204 No Content
```

## Impact
Minor REST convention violation. Clients expecting 204 on delete may mishandle the response.
