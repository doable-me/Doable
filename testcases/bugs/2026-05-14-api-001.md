# BUG-API-001: GET /health/ trailing slash returns 308 redirect instead of 200

**Date:** 2026-05-14
**Severity:** medium
**Area:** TC-API-HEALTH-001, TC-API-HEALTH-015

## Summary
`GET /health/` (with trailing slash) returns a 308 Permanent Redirect to `http://dev-api.doable.me/health` (note: http not https). `GET /health` (no slash) returns 200 correctly. This inconsistency means any client using a trailing slash gets redirected to an insecure HTTP URL.

## Steps to Reproduce
```
curl -si https://dev-api.doable.me/health/
```

## Actual Response
```
HTTP/1.1 308 Permanent Redirect
location: http://dev-api.doable.me/health
```

## Expected
200 with health JSON body (same as `/health`).

## Additional Impact
- `POST /health/` also returns 308 instead of 404
- `GET /health/extra-path` returns 401 instead of 404 (auth middleware intercepts before route matching)
- The redirect location uses HTTP not HTTPS — any follow would downgrade connection
