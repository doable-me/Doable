# BUG-2026-05-14-publish-002: Published project has null subdomain and published_url in API response

**Severity:** High  
**Environment:** dev (https://dev.doable.me)  
**Date:** 2026-05-14  
**Affected TCs:** TC-PUBLISH-SUBDOMAIN-001, TC-PUBLISH-LIFECYCLE-025

## Summary
A project with `status: "published"` returns `subdomain: null` and `published_url: null` from `GET /projects/{id}`. The published state is tracked but the subdomain/URL fields are not persisted.

## Steps to Reproduce
1. Log in as owner-pro@doable.me
2. Call `GET /projects/732bc993-0d20-4ce9-a0cb-9442cb3a6060`

## Expected
```json
{
  "status": "published",
  "subdomain": "smoke-vite",
  "published_url": "https://dev-smoke-vite.doable.me"
}
```

## Actual
```json
{
  "status": "published",
  "subdomain": null,
  "published_url": null
}
```

## Impact
- Dashboard cannot show live URL chip for published projects
- Users cannot find their published site URL from the API
- The `status: "published"` is misleading since no live URL exists

## Notes
The project `slug` field is populated ("smoke-vite") but `subdomain` is null. The deploy failure (BUG-001) may be the root cause — if deploys never succeed, subdomain/published_url never get written.
