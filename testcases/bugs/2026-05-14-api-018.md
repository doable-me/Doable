# BUG-API-018: GET /folders missing workspaceId returns 200 instead of 400

**Date:** 2026-05-14
**Severity:** medium
**Area:** TC-API-FOLDERS-002

## Summary
`GET /folders` without `?workspaceId=` query parameter returns 200 with empty data instead of 400. The workspaceId is required to scope the folder listing.

## Steps to Reproduce
```bash
curl "https://dev-api.doable.me/folders" \
  -H "Authorization: Bearer $TOKEN"
# HTTP 200: {"data":[]}
```

## Expected
```
HTTP/1.1 400 Bad Request
{"error":"workspaceId query parameter is required"}
```

## Impact
Silent empty result when workspaceId is missing, rather than a clear validation error. Can mask bugs in client code.
