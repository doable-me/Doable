# BUG-ADMIN-012: /admin crashes for every platform admin — "TypeError: A.map is not a function"

**Date:** 2026-05-15
**Severity:** P0 — entire /admin surface unreachable on dev and every fresh setup-server.sh install
**Status:** FIXED
**Reporter:** /ralph R17
**Related:** BUG-ADMIN-005 (the regression that introduced this)

## Summary

After commit `05b622dc` ("enrich GET /admin/users with plan, AI config, and credit fields"), every visit to `/admin` by a platform admin renders the Dashboard error boundary with:

> Something went wrong
>
> A.map is not a function
>
> Try again

The admin panel never renders — Feature Flags, Users & AI, DNS, Plans, Tools etc. are all unreachable.

## Reproduction (dev)

1. Log in to https://dev.doable.me as `admin2@doable.me` (TestPass123!) — confirmed `isPlatformAdmin: true`.
2. Navigate to https://dev.doable.me/admin.
3. Observe the error boundary above. Console shows
   `TypeError: A.map is not a function` originating from the minified admin page chunk.

## Root cause

`services/api/src/routes/admin-users.ts` GET /users returns
`{ data: [...], total, limit, offset }` (paginated envelope, camelCase keys).

`apps/web/src/hooks/use-platform-admin.ts:67` stores the whole response
unconditionally:

```ts
const data = await apiFetch("/admin/users");
setUsers(data); // <-- sets state to the envelope object
```

`apps/web/src/app/(dashboard)/admin/page.tsx:241` then computes
`displayUsers` via `users.map(...)`. Calling `.map` on an object instead of
an array throws synchronously during render and the (dashboard) error
boundary catches it.

A secondary mismatch made the recovery surface incorrect even if the
envelope had been unwrapped: the API uses camelCase
(`displayName`, `isPlatformAdmin`, `platformRole`, `createdAt`,
`workspaceId`) while every consumer (`AdminUser` type,
`admin-components.tsx`, `user-management-panel.tsx`,
`admin/page.tsx`) reads snake_case
(`display_name`, `is_platform_admin`, `platform_role`, …).

## Fix

1. `services/api/src/routes/admin-users.ts`: revert GET /users to a flat
   snake_case array (the shape every consumer already expects). Enriched
   fields `plan`, `ai_source`, `model`, `daily_credits`,
   `monthly_credits`, `rollover_credits`, `workspace_id` retained in
   snake_case. Pagination metadata dropped — no consumer ever read it.
2. `apps/web/src/hooks/use-platform-admin.ts`: `loadFeatures` and
   `loadUsers` now unwrap an optional `{ data: [...] }` envelope and
   assert `Array.isArray` before setting state. Future drift can't
   produce the same crash class.
3. `scripts/r17-test-admin-users-contract.ts`: standalone tsx probe that
   pins the contract — fails if a future patch ships an envelope or
   camelCase keys. Run with `pnpm exec tsx scripts/r17-test-admin-users-contract.ts`.

## Why it slipped past CI

There was no automated test for `/admin` page render or `/admin/users`
response shape. The enrichment commit changed the response shape with
TypeScript still happy (the hook used implicit `any`), so the contract
break only surfaced at runtime in the React render of the admin page.

## Prevention

The new contract probe under `scripts/` runs against the actual handler
module so future enrichment commits cannot silently flip the wire shape
again. Pair it with the existing claude-in-chrome smoke for `/admin`
post-deploy (still recommended as the human-facing acceptance check).
