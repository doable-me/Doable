# BUG-API-019: POST /deploy/:projectId/publish returns 500 for empty projects

**Date:** 2026-05-14
**Severity:** high
**Area:** TC-API-DEPLOY-001

## Summary
`POST /deploy/:projectId/publish` returns 500 Internal Server Error when the project has no files (empty project). The build system crashes with "Cannot resolve entry module index.html" and returns errorCode "build_failed_compile". This should return a clean 4xx (e.g. 422 Unprocessable) rather than 500.

## Steps to Reproduce
```bash
curl -X POST "https://dev-api.doable.me/deploy/<empty-project-id>/publish" \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{}'
```

## Actual Response
```
HTTP/1.1 500 Internal Server Error
{"error":"Deployment failed","errorCode":"build_failed_compile","data":{"buildLog":"... [UNRESOLVED_ENTRY] Cannot resolve entry module index.html...","errorMessage":"Build exited with code 1"}}
```

## Expected
```
HTTP/1.1 422 Unprocessable Entity
{"error":"Project has no files to deploy"}
```
or 202 with later status=failed (acceptable per TC-API-DEPLOY-017).

## Impact
500 errors are alarming in monitoring systems. Empty project deploy should be gracefully rejected, not crash with 500.
