# BUG-2026-05-14-publish-001: All deploys fail — /var/lib/doable-sites missing on dev server

**Severity:** Critical  
**Environment:** dev (https://dev.doable.me)  
**Date:** 2026-05-14  
**Affected TCs:** TC-DEPLOY-LIFECYCLE-001, TC-PUBLISH-LIFECYCLE-001, TC-PUBLISH-SUBDOMAIN-004

## Summary
Both "Deploy to Live" and "Deploy to Test" fail on the dev server. The deploy pipeline reaches the file-writing stage but the target directory `/var/lib/doable-sites` does not exist on the server, causing an ENOENT error.

## Steps to Reproduce
1. Log in as owner-pro@doable.me
2. Open any project in the editor (e.g. Smoke Vite Updated, project ID 732bc993-0d20-4ce9-a0cb-9442cb3a6060)
3. Click the "Deploy" button in the editor toolbar
4. Select Live environment, click "Deploy to Live"

## Expected
Deployment succeeds: project files written to `/var/lib/doable-sites/<subdomain>/live`, Caddy config updated, site live at `dev-<slug>.doable.me`.

## Actual
Deploy fails with error:
```
Failed to deploy to /var/lib/doable-sites/dev-vite-updated-zpnwx/live: ENOENT: no such file or directory, mkdir '/var/lib/doable-sites'
```

Same failure for "Deploy to Test" (preview deploy).

## Network Evidence
- UI calls: `POST https://dev-api.doable.me/projects/{id}/publish`
- Network shows status 404 for this endpoint
- UI displays the ENOENT filesystem error (stale/server-side error)

## Root Cause (Suspected)
The `/var/lib/doable-sites` directory was not created during dev server provisioning. Run `mkdir -p /var/lib/doable-sites` on the dev server and ensure the API process has write permissions.

## Secondary Issue
The UI shows a raw server-side filesystem error message to end users instead of a friendly message. The error "ENOENT: no such file or directory, mkdir '/var/lib/doable-sites'" leaks internal server path information and is not user-friendly.
