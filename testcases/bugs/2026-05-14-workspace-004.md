# BUG-2026-05-14-WS-004: dev-api.doable.me persistent 502 from ~17:42 UTC

**Date:** 2026-05-14
**Environment:** dev (dev-api.doable.me)
**Severity:** Critical (blocks all testing)
**Category:** Infrastructure / Service Outage

## Summary
The dev API server (`dev-api.doable.me`) began returning `502 Bad Gateway` for ALL endpoints from approximately 17:42 UTC on 2026-05-14. This includes unauthenticated endpoints (`/auth/login`, `/health`) and persisted for at least 30+ minutes.

## Timeline
- 17:33 UTC: API functional — workspace/project CRUD tests passing
- 17:41 UTC: First 502 observed on `/projects/shared`
- 17:42 UTC: All endpoints returning 502 including `/auth/login` and `/health`
- 17:57 UTC: Still 502 after 15s retry
- 18:02 UTC: Still 502 after 60s retry

## Evidence
```
curl -s https://dev-api.doable.me/auth/login -> HTTP 502
curl -s https://dev-api.doable.me/health -> HTTP 502
```
CF-RAY: 9fbbb35a39b27aff-MAA (Cloudflare upstream error)

## Likely Cause
The `/projects/shared` route crash (BUG-2026-05-14-WS-003) may have caused the API process to crash entirely (uncaught exception / OOM), taking down the entire service.

## Impact
- All QA testing blocked from 17:42 UTC onward
- Following test cases could not be executed: TC-WS-ROLE-003 through TC-WS-ROLE-040, TC-WS-MEM-005 through TC-WS-MEM-017, TC-WS-CRUD-047/051/053/054, TC-PROJ-COLLAB-005 through TC-PROJ-COLLAB-020, TC-PROJ-DELETE-029, TC-PROJ-LIST-044/045

## Recommended Action
Restart the API tmux window on dev server. Investigate crash logs. Fix the /projects/shared route that likely triggered the crash.
