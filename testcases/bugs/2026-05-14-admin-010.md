# BUG-ADMIN-010: API admin endpoints missing Cache-Control: no-store header

**Date:** 2026-05-14
**Severity:** Medium / P0 (security)
**Test Cases:** TC-ADMIN-DASHBOARD-025

## Summary
Admin API endpoints (e.g., GET /admin/projects) do not return a `Cache-Control: no-store, private` header. Per TC-ADMIN-DASHBOARD-025, admin responses must have `Cache-Control: no-store, private` to prevent proxy/browser caching of sensitive admin data.

## Steps to Reproduce
```
curl -I -H "Authorization: Bearer <admin-token>" https://dev-api.doable.me/admin/projects
```

## Expected Headers (per spec)
```
Cache-Control: no-store, private
X-Frame-Options: DENY
Referrer-Policy: same-origin
Content-Security-Policy: (with nonce)
```

## Actual Headers
```
referrer-policy: no-referrer
strict-transport-security: max-age=15552000; includeSubDomains
x-content-type-options: nosniff
x-frame-options: SAMEORIGIN
```

Missing: `Cache-Control: no-store`
Wrong: `x-frame-options: SAMEORIGIN` (should be `DENY` for admin)
Missing: `Content-Security-Policy`

## Impact
Admin responses may be cached by proxies or browsers. Shared/corporate proxies could serve cached admin data to subsequent users. `SAMEORIGIN` vs `DENY` allows the admin panel to be embedded in iframes from the same origin.
