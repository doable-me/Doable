# BUG-ADMIN-007: DELETE /admin/features/:key permanently deletes feature with no confirmation — data loss

**Date:** 2026-05-14
**Severity:** Critical / P0
**Test Cases:** TC-ADMIN-FF-010, TC-ADMIN-FF-022

## Summary
DELETE /admin/features/:key permanently deletes feature flags from the database with no confirmation dialog, no soft-delete, and no audit trail requirement. Features `ai_chat` and `analytics` were permanently deleted during testing. The feature list dropped from 15 to 13 items. Restore via POST /admin/features was not possible (endpoint rejected payload).

## Steps to Reproduce
```
DELETE https://dev-api.doable.me/admin/features/analytics
Authorization: Bearer <admin-token>
```
Response: `{"ok": true}` HTTP 200

## Expected
Per TC-ADMIN-FF-022: Deletion should NOT be allowed. Admin should only be able to disable/toggle flags, not delete them, to preserve audit history. At minimum, should require 2-step confirmation and write audit row.

## Actual
Feature is permanently deleted immediately. Feature count went from 15 → 13 after two DELETE calls.

## Evidence
- Before: 15 features including `ai_chat` (enabled=true) and `analytics` (enabled=true)
- After DELETE ai_chat + analytics: only 13 features, both gone
- POST /admin/features to restore: returned 400 (restore impossible via API)
- Final feature list: `['ai_settings', 'billing', 'code_editor', 'connectors', 'custom_domains', 'github_sync', 'publish', 'security_center', 'templates', 'version_history', 'visual_editor', 'workspace_members', 'workspaces']`

## Impact
Catastrophic data loss. Platform-critical feature flags can be permanently deleted by any platform admin with a single HTTP call. No recovery path available via API. The `ai_chat` feature flag controls AI chat access across the platform.
