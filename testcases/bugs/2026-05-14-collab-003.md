# BUG-COLLAB-003: Login shows generic error when API is down (502/503)

**Date**: 2026-05-14
**Environment**: dev (dev.doable.me)
**Severity**: MEDIUM
**Area**: Authentication UX / Error Handling

## Summary
When the API server returns 502 Bad Gateway during login, the frontend displays a generic "Something went wrong. Please try again." message instead of a more informative error like "Service temporarily unavailable" or "Unable to connect to server."

## Steps to Reproduce
1. Ensure dev-api.doable.me is returning 502
2. Navigate to dev.doable.me/login
3. Enter valid credentials (owner-pro@doable.me / TestPass123!)
4. Click Sign in

## Expected
Error message should communicate the nature of the failure:
- "Service temporarily unavailable. Please try again in a few minutes."
- Or at minimum distinguish between "wrong credentials" vs "server error"

## Actual
- Displays: "Something went wrong. Please try again."
- Same message shown for 429 rate limiting, 502 server error, and potentially invalid credentials
- User cannot distinguish between their error and a server error

## Evidence
- Playwright snapshot: `generic[ref=e52]: "Something went wrong. Please try again."`
- Chrome screenshot: Red banner with "Something went wrong. Please try again."
- Network: POST /auth/login → 502 (confirmed via PowerShell)

## Impact
Users cannot tell if login failure is due to wrong password, rate limiting, or infrastructure outage. This leads to repeated failed attempts, making rate limiting worse.
