# BUG-ADMIN-006: PATCH /admin/features/ai_chat returns 404 — feature key lookup broken for underscore keys

**Date:** 2026-05-14
**Severity:** Critical
**Test Cases:** TC-ADMIN-FF-002, TC-ADMIN-FF-003, TC-ADMIN-DASHBOARD-045, TC-ADMIN-DASHBOARD-046

## Summary
PATCH /admin/features/ai_chat returns 404 "Feature not found" even though `ai_chat` appears in GET /admin/features list. The individual feature lookup by key is broken for feature keys containing underscores. Other keys without underscores (analytics, billing, publish, templates) work fine.

## Steps to Reproduce
```
PATCH https://dev-api.doable.me/admin/features/ai_chat
Authorization: Bearer <admin-token>
Content-Type: application/json
{"enabled": true}
```

## Expected
200 with updated feature state.

## Actual
```json
{"error": "Feature not found"}  HTTP 404
```

Also broken:
- GET /admin/features/ai_chat → 404
- GET /admin/features/ai-chat → 404 (hyphen variant also fails)

Working:
- PATCH /admin/features/analytics → 200
- PATCH /admin/features/billing → 200

## Root Cause Hypothesis
Route parameter matching or DB lookup may be URL-decoding or transforming underscores. The key `ai_chat` may be stored differently in DB vs how the route parameter is parsed.

## Note
This bug contributed to accidental permanent deletion of ai_chat (see BUG-ADMIN-007).
