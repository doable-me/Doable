# BUG-2026-05-14-BILLING-004: /billing/subscription endpoint returns 404

**Severity:** High
**Area:** Billing API
**Endpoint:** GET /billing/subscription
**Discovered:** 2026-05-14

## Description
The documented endpoint GET /billing/subscription returns 404 Not Found. The test case TC-BILLING-PLANS-012 expects a 200 with {plan, status, currentPeriodEnd, cancelAtPeriodEnd}.

## Steps to Reproduce
```
GET https://dev-api.doable.me/billing/subscription
Authorization: Bearer {valid_token}
```

## Expected
```json
{"plan": "pro", "status": "active", "currentPeriodEnd": "...", "cancelAtPeriodEnd": false}
```

## Actual
```json
{"error": "Not Found", "path": "/billing/subscription"}
```

## Also Missing (404)
- GET /billing/limits — 404
- POST /billing/cancel — 404
- POST /billing/upgrade — 404 (subscribe endpoint works instead)
- GET /billing/topup/history — 404
- POST /billing/grant — 404
- POST /billing/revoke — 404
