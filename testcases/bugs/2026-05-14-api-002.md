# BUG-API-002: POST /projects accepts invalid framework enum values

**Date:** 2026-05-14
**Severity:** high
**Area:** TC-API-PROJECTS-021

## Summary
`POST /projects` with `framework:"cobol"` (not a valid enum) returns 201 Created instead of 400. The project is stored with `framework_id:"vite-react"` — the invalid value is silently defaulted without any validation error.

## Steps to Reproduce
```bash
curl -X POST https://dev-api.doable.me/projects \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"name":"CobolTest","framework":"cobol","workspaceId":"<wid>"}'
```

## Actual Response
```json
HTTP/1.1 201 Created
{"data":{"id":"b4904a3c-...","framework_id":"vite-react","name":"CobolTest",...}}
```

## Expected
```json
HTTP/1.1 400 Bad Request
{"error":"Validation failed","details":{"framework":["Invalid enum value"]}}
```

## Impact
Invalid framework values are silently accepted and stored with a default, making it impossible to rely on framework_id for routing/building logic. Projects created this way may behave unexpectedly.
