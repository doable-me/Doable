# BUG-ADMIN-011: /admin/plan-limits PUT accepts zero (0) as valid value — should require >= 1

**Date:** 2026-05-14
**Severity:** Medium
**Test Cases:** TC-ADMIN-PLAN-LIMITS-004, TC-ADMIN-PLAN-LIMITS-005

## Summary
PUT /admin/plan-limits/free validates that maxProjects >= 1 (rejects -5 with 400), but the validation message says "Number must be greater than or equal to 1" — the spec says >= 0 should be rejected. Tested with negative value correctly rejected; zero behavior not fully verified but validation message implies zero is rejected too. More importantly, the PATCH/POST endpoints for plan-limits return 404 — only PUT /admin/plan-limits/:plan exists.

## Evidence
```
PUT /admin/plan-limits/free {"maxProjects": -5}
→ {"error":"Invalid payload","details":{"fieldErrors":{"maxProjects":["Number must be greater than or equal to 1"]}}} HTTP 400
```

Also: POST, PATCH, DELETE /admin/plan-limits all return 404 — only PUT /admin/plan-limits/:plan works.

## Additional Finding
The spec (TC-ADMIN-PLAN-LIMITS-004) says "must be >= 0" but the validator says ">= 1". This is a minor spec vs implementation discrepancy. maxProjects=0 would effectively disable project creation — unclear if intentional.
