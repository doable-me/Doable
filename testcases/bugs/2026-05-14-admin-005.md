# BUG-ADMIN-005: /admin/users missing plan, AI config, and credit fields

**Date:** 2026-05-14
**Severity:** Critical
**Test Cases:** TC-ADMIN-DASHBOARD-036, TC-ADMIN-DASHBOARD-037, TC-ADMIN-DASHBOARD-038

## Summary
GET /admin/users returns only 6 fields per user: `id, email, display_name, is_platform_admin, platform_role, created_at`. The following fields required by the test spec are completely absent: `plan`, `ai_source`, `model`, `daily_credits`, `monthly_credits`, `rollover_credits`. All 28 users are missing these fields.

## Steps to Reproduce
```
GET https://dev-api.doable.me/admin/users
Authorization: Bearer <admin-token>
```

## Expected
Each user row includes: email, display_name, role, plan, ai_source, model, daily_credits, monthly_credits, rollover_credits per TC-ADMIN-DASHBOARD-036.

## Actual
```json
{
  "id": "a3aaa53c-fd7a-4c8f-9d8f-5dc70f8d9f8f",
  "email": "shriyan@altrosyn.com",
  "display_name": "shriyan S",
  "is_platform_admin": false,
  "platform_role": "member",
  "created_at": "2026-05-14T13:15:34.334Z"
}
```
No `plan`, `ai_source`, `model`, `daily_credits`, `monthly_credits` fields. All 28 users affected.

## Impact
Admin Users & AI tab cannot show AI config, plan, or credit info per user. Admins cannot see user plan/credits from the user list.
