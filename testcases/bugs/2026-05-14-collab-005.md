# BUG-COLLAB-005: /projects/{id}/runtime/metrics returns 404

**Date**: 2026-05-14
**Environment**: dev (dev-api.doable.me)
**Severity**: MEDIUM
**Area**: API / Runtime Metrics

## Summary
The runtime metrics endpoint `GET /projects/{projectId}/runtime/metrics` returns HTTP 404, indicating the endpoint is either not implemented or not registered in the API router.

## Steps to Reproduce
1. Open any project in the editor
2. Observe browser console for metrics requests

## Expected
`GET /projects/{id}/runtime/metrics` → 200 with runtime metrics data

## Actual
`GET https://dev-api.doable.me/projects/f42fcc90-7789-465f-853c-d05ef54c5915/runtime/metrics` → 404

## Evidence
Browser console (all=true, level=error):
- `Failed to load resource: 404 () @ https://dev-api.doable.me/projects/f42fcc90-7789-465f-853c-d05ef54c5915/runtime/metrics`
- Error repeats continuously (50+ times), suggesting client keeps polling without respecting 404 as terminal error

## Secondary Issue
Client continues polling a 404 endpoint in a tight loop — this is a secondary bug. The client should stop polling after receiving 404 (not a transient error).

## Impact
Runtime metrics panel in editor is non-functional. Excessive polling on 404 wastes bandwidth.
