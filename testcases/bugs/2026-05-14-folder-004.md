# BUG-FOLDER-004: GET /workspaces/:id/folders returns 200 for unauthorized workspace

**Date**: 2026-05-14
**Severity**: High
**Feature**: Folders — workspace authorization on list endpoint
**Environment**: dev (https://dev-api.doable.me)

## Steps to Reproduce

1. Authenticate as owner-pro@doable.me (member of workspace A, NOT member of workspace B)
2. GET /workspaces/{workspaceBId}/folders

## Expected

403 Forbidden — user is not a member of workspace B

## Actual

200 OK with empty array — no authorization check; server returns empty results instead of rejecting the request

## Impact

Information disclosure: any authenticated user can probe any workspace's folder endpoint. While the result is currently empty (no data leak), this represents a missing authorization layer that could expose data if the query is incomplete.
