# BUG-API-023: GET /projects?limit=-1 returns 200 instead of 400

**Date:** 2026-05-14
**Severity:** medium
**Area:** TC-API-PROJECTS-015

## Summary
`GET /projects?workspaceId=<wid>&limit=-1` returns 200 instead of 400. Negative limit values should be rejected by input validation.

## Steps to Reproduce
```bash
curl "https://dev-api.doable.me/projects?workspaceId=<wid>&limit=-1" \
  -H "Authorization: Bearer $TOKEN"
# HTTP 200 OK (returns results)
```

## Expected
```
HTTP/1.1 400 Bad Request
{"error":"Validation failed","details":{"limit":["Must be positive"]}}
```

## Impact
Clients can pass invalid pagination parameters without error feedback. May cause unexpected behavior in DB queries.
