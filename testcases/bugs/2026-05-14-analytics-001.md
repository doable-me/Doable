# BUG-2026-05-14-analytics-001: Analytics dashboard endpoints lack project membership authorization

## Severity
P0 — Data isolation failure: any authenticated user can read another user's project analytics

## Test Case
TC-ANALYTICS-DASHBOARD-019

## Environment
- Target: https://dev-api.doable.me
- Date: 2026-05-14

## Steps to Reproduce
1. Authenticate as admin2 (user: admin2@doable.me, NOT a member of owner-pro's QA Team Workspace)
2. GET `https://dev-api.doable.me/analytics/projects/80bd2376-2c67-4df8-b459-ad4e8dce0c81/overview`
   - Header: `Authorization: Bearer <admin2-token>`

## Expected
HTTP 403 or 404 — admin2 is not a member of the workspace that owns this project

## Actual
HTTP 200 with full analytics data:
```json
{"data":{"visitors":0,"pageViews":0,"sessions":0,"avgDuration":0,"bounceRate":0,"changes":{"visitors":0,"pageViews":0,"sessions":0,"avgDuration":0,"bounceRate":0}}}
```

## Root Cause (code)
File: `services/api/src/routes/analytics/dashboard.ts`

The `dashboardRoutes` only applies `authMiddleware` via `dashboardRoutes.use("/projects/*", authMiddleware)` — this confirms the user is logged in, but does NOT verify the user is a member of the workspace that owns the project.

Each handler does `projects.findById(projectId)` and returns 404 if project not found, but never checks `workspace_id` membership for the requesting user. Any authenticated user who knows (or guesses) a project UUID can read its full analytics.

Affected endpoints (all under `/analytics/projects/:id/*`):
- `/overview`, `/timeseries`, `/pageviews`, `/events`, `/pages`, `/referrers`, `/devices`, `/browsers`, `/os`, `/realtime`, `/settings`
- PUT `/settings` (can also toggle analytics on/off for any project)

## Fix Direction
After resolving the project, verify the requesting user has a role in `project.workspace_id`:
```typescript
const membership = await workspaces.getMemberRole(project.workspace_id, userId);
if (!membership) return c.json({ error: "Forbidden" }, 403);
```
Or add a shared middleware that wraps all `/projects/:id/*` analytics routes.
