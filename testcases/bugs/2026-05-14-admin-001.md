# BUG-ADMIN-001: /admin/projects search filter not applied — all projects returned regardless of query

**Date:** 2026-05-14
**Severity:** Critical
**Test Cases:** TC-ADMIN-PROJECTS-004, TC-ADMIN-PROJECTS-005, TC-ADMIN-PROJECTS-007, TC-ADMIN-PROJECTS-008

## Summary
GET /admin/projects?q=<term> ignores the search query and returns all 48 projects regardless of the search term. Even a nonsense query like `?q=nonexistentzxqwerty999` returns all 48 rows. Filtering is completely non-functional.

## Steps to Reproduce
```
GET https://dev-api.doable.me/admin/projects?q=smoke
Authorization: Bearer <admin-token>
```

## Expected
Only projects whose name, owner email, or workspace name contains "smoke" (case-insensitive). Should return ~1-3 matching rows.

## Actual
Returns all 48 projects. Query param `q` is silently ignored.

## Evidence
- `?q=smoke` → 48 results (all projects)
- `?q=SMOKE` → 48 results
- `?q=nonexistentzxqwerty999` → 48 results
- `?q=%25` → 48 results (SQL wildcard not exploited, just ignored)

## Impact
Admin cannot search/filter projects. The search box in the admin UI is non-functional. For large platforms with thousands of projects this makes the admin panel unusable for project lookup.
