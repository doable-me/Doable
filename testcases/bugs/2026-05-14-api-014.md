# BUG-API-014: GET /integrations/admin/pieces returns 404 instead of 403 for non-admin

**Date:** 2026-05-14
**Severity:** medium
**Area:** TC-API-INTEG-027

## Summary
`GET /integrations/admin/pieces` with a non-admin token returns 404 instead of 403. This leaks that the endpoint doesn't exist at the path (or is not mounted), rather than giving a proper authorization error.

## Steps to Reproduce
```bash
curl "https://dev-api.doable.me/integrations/admin/pieces" \
  -H "Authorization: Bearer $NON_ADMIN_TOKEN"
# HTTP 404 Not Found
```

## Expected
403 Forbidden (non-admin access denied), not 404.

## Impact
Security-through-obscurity issue. Non-admins see 404 suggesting the route doesn't exist rather than 403 (access denied). Could also indicate the route is not properly mounted.
