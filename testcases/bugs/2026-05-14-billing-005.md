# BUG-2026-05-14-BILLING-005: /billing/webhook returns 502 Bad Gateway for most POST requests

**Severity:** High  
**Area:** Billing API — Webhook handler
**Endpoint:** POST /billing/webhook
**Discovered:** 2026-05-14

## Description
POST /billing/webhook returns 502 Bad Gateway when called from outside (via Cloudflare tunnel). Only the signature-validation rejection returns a proper 400. Any attempt to send a webhook event body results in 502.

## Steps to Reproduce
```
POST https://dev-api.doable.me/billing/webhook
Content-Type: application/json
Body: {"type":"checkout.session.completed","id":"evt_test"}
```

## Expected
- TC-BILLING-WEBHOOK-002: 400 "Missing stripe-signature header" ✓ (this works)
- TC-BILLING-WEBHOOK-003: 400 "Webhook verification failed" ✓ (this works)  
- TC-BILLING-WEBHOOK-018: With bypass mode, should return 200 with skip

## Actual
- 502 Bad Gateway from Cloudflare for most webhook scenarios
- Note: Missing-sig and invalid-sig paths (pre-body-parse) work, but actual event processing path returns 502

## Impact
Stripe webhook delivery will fail. Subscription activations, topup grants, plan updates from Stripe will not be processed.
