# BUG-021: Usage log missing token and model data

**Date:** 2026-05-14
**Suite:** 05-ai-chat
**Severity:** Medium
**Test Cases:** TC-AI-CHAT-CREDITS-023

## Description
The `GET /billing/usage` endpoint returns usage log entries where `prompt_tokens`, `completion_tokens`, and `model` are all `null`, despite the SSE stream's `usage` event containing full token counts and model name.

## Evidence
SSE usage event:
```json
{"type":"usage","data":{"promptTokens":17614,"completionTokens":35,"model":"MiniMax-M2.7-highspeed",...}}
```

Usage log entry:
```json
{"credits_consumed":1,"action_type":"chat_message","prompt_tokens":null,"completion_tokens":null,"model":null,...}
```

## Expected
Usage log rows populated with `prompt_tokens`, `completion_tokens`, and `model` from the completed stream.

## Impact
Admins cannot audit token consumption per model. Cost analysis, billing verification, and per-model usage reporting are broken. The spec requires `provider, model, prompt_tokens, completion_tokens, total_tokens, cost_units` in the log.
