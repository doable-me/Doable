# BUG-ADMIN-008: Multiple admin routes missing — chat, trace, audit, moderation, runtime, impersonation endpoints all 404

**Date:** 2026-05-14
**Severity:** Critical
**Test Cases:** TC-ADMIN-CHAT-001, TC-ADMIN-TRACE-001, TC-ADMIN-AUDIT-001, TC-ADMIN-MODERATION-001, TC-ADMIN-RUNTIME-001, TC-ADMIN-IMP-001

## Summary
The following admin API routes specified in the test suite all return 404. Entire feature areas of the admin panel are missing from the API.

## Missing Routes (all return 404)

| Route | Expected By | Purpose |
|---|---|---|
| GET /admin/chat | TC-ADMIN-CHAT-001 | Browse all AI sessions |
| GET /admin/trace | TC-ADMIN-TRACE-001 | OTel trace search |
| GET /admin/audit | TC-ADMIN-AUDIT-001 | Activity events list |
| GET /admin/moderation | TC-ADMIN-MODERATION-001 | Moderation queue |
| GET /admin/runtime | TC-ADMIN-RUNTIME-001 | Systemd unit management |
| GET /admin/impersonate | TC-ADMIN-IMP-001 | User impersonation |
| POST /api/otlp/v1/traces | TC-ADMIN-TRACE-003 | OTLP ingestion |

## Present Routes (working)
- GET /admin/projects ✓
- GET /admin/users ✓
- GET /admin/plan-limits ✓
- GET /admin/dev-servers ✓
- GET /admin/features ✓
- GET /admin/audit/actions ✓
- GET /admin/marketplace/reports ✓
- GET /admin/marketplace/moderation/queue ✓

## Impact
~60% of admin panel functionality is unavailable: chat review, trace/xray, audit events, moderation actions, runtime management, and impersonation are completely absent from the API.
