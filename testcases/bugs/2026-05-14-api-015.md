# BUG-API-015: GET /ai/provider-catalog requires auth (spec says public)

**Date:** 2026-05-14
**Severity:** medium
**Area:** TC-API-PROV-001

## Summary
`GET /ai/provider-catalog` returns 401 Unauthorized when called without auth. The test spec explicitly states this should be a public endpoint (no auth required), with ETag caching.

## Steps to Reproduce
```bash
curl "https://dev-api.doable.me/ai/provider-catalog"
# HTTP 401 Unauthorized

curl "https://dev-api.doable.me/ai/provider-catalog" \
  -H "Authorization: Bearer $TOKEN"
# HTTP 200 OK (works with auth)
```

## Expected
200 without auth. ETag caching headers present.

## Impact
Third-party clients and unauthenticated UI bootstrap cannot fetch the provider catalog. ETag caching (304 support, which works with auth) also becomes inaccessible publicly.
