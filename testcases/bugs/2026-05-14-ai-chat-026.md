# BUG-026: owner-pro user has viewer role in own workspace — cannot use AI chat

**Date:** 2026-05-14
**Suite:** 05-ai-chat
**Severity:** High
**Test Cases:** TC-AI-CHAT-SEND-001 (blocked for primary test token)

## Description
The `owner-pro@doable.me` test user is the `owner_id` on workspace `492bdda5-...` (pro plan) but has `userRole: viewer` when querying the workspace. This means the primary test token provided for AI chat testing cannot send messages because the `requireCredits` + role check blocks viewers.

## Evidence
```
GET /workspaces/492bdda5-6f66-46c7-a6e6-22a8b391fa9d
=> {"owner_id":"7b36f2e7-...","userRole":"viewer"}

POST /projects/a9bcb1a9-.../chat {"content":"hello","mode":"agent"}
=> HTTP 403 {"error":"Viewers cannot use AI chat"}
```

The `workspace_members` table has `role: viewer` for this user despite being `owner_id` in the workspaces table — a data integrity mismatch.

## Expected
Workspace owner should have `owner` or `admin` role in `workspace_members`, never `viewer`.

## Impact
The provided test token is unusable for AI chat testing. All primary test scenarios requiring the pro-plan token had to be run with `admin2` (platform admin) which bypasses normal role enforcement. Test coverage for pro-plan credit behavior is limited.
