# BUG-2026-05-14-BILLING-002: CRITICAL - Cross-tenant billing balance data leak

**Severity:** Critical (Security)
**Area:** Billing API — /billing/balance
**Endpoint:** GET /billing/balance?workspaceId={id}
**Discovered:** 2026-05-14

## Description
A user authenticated as owner-free@doable.me can read the billing balance of owner-pro@doable.me's workspace by supplying the pro workspace ID in the query parameter. The API returns 200 with full pro plan billing data instead of 403.

## Steps to Reproduce
1. Obtain a valid token for owner-free@doable.me
2. GET /billing/balance?workspaceId=492bdda5-6f66-46c7-a6e6-22a8b391fa9d (pro workspace)
   with Authorization: Bearer {free_user_token}
3. Observe 200 response with pro plan data

## Expected
- 403 Forbidden — user does not have access to this workspace

## Actual
```json
HTTP 200
{"data":{"dailyRemaining":50,"dailyMax":50,"monthlyRemaining":500,"monthlyMax":500,"topupRemaining":0,"planUnlimited":false,"planType":"pro"}}
```

## Impact
Any authenticated user can enumerate billing data for any workspace if they know the workspace UUID. This exposes plan type, credit usage, and remaining balances.

## Also Affected
- GET /billing/invoices?workspaceId={other_workspace} — returns 200 with empty array (no 403)
