# BUG-2026-05-14-MCP-005 — Integration catalog search (?q=) not working

**Date:** 2026-05-14
**Env:** dev (dev-api.doable.me)
**Severity:** MEDIUM
**Status:** OPEN
**Test Case:** TC-INTEG-LIST-006

## Description

The `?q=` search parameter on /integrations/catalog does not filter results. The `?search=` parameter works correctly.

## Evidence

```
GET /integrations/catalog?q=stripe → Returns ActiveCampaign, Activepieces, Actual Budget (wrong - returns all, no filter)
GET /integrations/catalog?search=stripe → Returns Moonclerk, Saastic, Stripe (correct)
```

## Expected

`?q=stripe` should filter by name/description and return Stripe as a top result (per TC-INTEG-LIST-006 spec which uses `?q=`).

## Impact

Any client using ?q= for search gets no filtering - all 533 integrations returned, causing poor UX.
