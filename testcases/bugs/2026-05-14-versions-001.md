# BUG-VER-001: GET /projects/:id/versions/auto causes 500 UUID parse error

**Date**: 2026-05-14
**Severity**: Medium
**Feature**: Versions — route conflict with literal "auto" segment
**Environment**: dev (https://dev-api.doable.me)

## Steps to Reproduce

1. Authenticate as owner-pro@doable.me
2. GET https://dev-api.doable.me/projects/{projectId}/versions/auto

## Expected

Either: 200 with the latest auto-generated version snapshot, OR 404 Not Found if the "auto" route is not implemented

## Actual

500 Internal Server Error — server attempts to parse the literal string "auto" as a UUID, which fails and throws an unhandled exception

## Root Cause (observed)

The route `/projects/:id/versions/:versionId` catches the "auto" segment and passes it to a UUID-based DB lookup without guard validation.

## Impact

Any client that probes or uses the "auto" alias crashes the request handler with a 500 instead of a clean error.
