# BUG-011: "chat" mode not implemented — only agent/plan/visual-edit accepted

**Date:** 2026-05-14
**Suite:** 05-ai-chat
**Severity:** High
**Test Cases:** TC-AI-CHAT-SEND-003, TC-AI-CHAT-MODES-003

## Description
The test spec and UI documentation reference a "chat" mode for AI conversations, but the API Zod schema only accepts `agent | plan | visual-edit`. Sending `mode:"chat"` returns HTTP 400 with a ZodError.

## Reproduction
```
POST https://dev-api.doable.me/projects/:id/chat
{"content":"explain monads","mode":"chat"}
```

## Actual Response
```json
HTTP 400
{"success":false,"error":{"issues":[{"received":"chat","code":"invalid_enum_value","options":["agent","plan","visual-edit"],"path":["mode"],"message":"Invalid enum value. Expected 'agent' | 'plan' | 'visual-edit', received 'chat'"}],"name":"ZodError"}}
```

## Expected
HTTP 200 with SSE stream in chat mode (no tool execution, conversational responses only).

## Impact
Any client sending mode:"chat" receives a 400 error. All TC-AI-CHAT-MODES-003 through 006 tests that reference chat mode are untestable.
