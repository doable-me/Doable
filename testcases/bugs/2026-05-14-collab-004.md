# BUG-COLLAB-004: Preview WebSocket returns 502 - live preview broken in dev

**Date**: 2026-05-14
**Environment**: dev (dev-api.doable.me)
**Severity**: HIGH
**Area**: Preview / WebSocket / Infrastructure

## Summary
The preview WebSocket endpoint `wss://dev-api.doable.me/preview/{projectId}/` returns HTTP 502 during the WebSocket handshake, making live preview completely non-functional in the dev environment.

## Steps to Reproduce
1. Open any project in the editor on dev.doable.me (when logged in)
2. Observe browser console
3. Preview panel fails to load

## Expected
WebSocket handshake should succeed (101 Switching Protocols), preview iframe should display project output.

## Actual
```
WebSocket connection to 'wss://dev-api.doable.me/preview/f42fcc90-7789-465f-853c-d05ef54c5915/' failed:
Error during WebSocket handshake: Unexpected response code: 502
```
This error repeats continuously with exponential backoff (observed 50+ times in console log).

## Evidence
Browser console (all=true): 100+ identical WebSocket 502 errors for project ID `f42fcc90-7789-465f-853c-d05ef54c5915`

## Root Cause
Likely same root cause as BUG-COLLAB-002 — API server is down, and the preview WebSocket is proxied through the same API server.

## Impact
Shared Preview feature (TC focus area #4) completely untestable. Both users cannot view live preview.
