# BUG-ADMIN-002: /admin/projects pagination not working — same page returned for all page numbers

**Date:** 2026-05-14
**Severity:** Critical
**Test Cases:** TC-ADMIN-PROJECTS-003, TC-ADMIN-PROJECTS-016, TC-ADMIN-PROJECTS-017, TC-ADMIN-PROJECTS-040

## Summary
GET /admin/projects?page=N&per=5 ignores the `page` and `per` query parameters. Every page returns all 48 projects starting from the same first item. Offset-based pagination is completely non-functional.

## Steps to Reproduce
```
GET https://dev-api.doable.me/admin/projects?page=1&per=5
GET https://dev-api.doable.me/admin/projects?page=2&per=5
```

## Expected
- page=1,per=5: returns 5 items (items 1-5)
- page=2,per=5: returns 5 items (items 6-10), first item differs from page 1

## Actual
Both pages return all 48 projects. Same first item ID on both pages:
`a9bcb1a9-20ea-4ad5-a4e3-1ed9662284ac`

Additionally:
- `meta` field is empty `{}` — no pagination metadata returned
- `?per=1000000` returns 48 rows with no X-Param-Clamped header (TC-ADMIN-PROJECTS-040)

## Evidence
- page=1 per=5: got 48 projects, first ID `a9bcb1a9-20ea-4ad5-a4e3-1ed9662284ac`
- page=2 per=5: got 48 projects, first ID `a9bcb1a9-20ea-4ad5-a4e3-1ed9662284ac`
- Meta: `{}`

## Impact
Admin cannot paginate through projects. All projects always shown, no page controls work.
