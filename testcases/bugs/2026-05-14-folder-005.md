# BUG-FOLDER-005: GET/PATCH/DELETE /folders/:id missing workspace membership check

**Date**: 2026-05-14
**Severity**: High
**Feature**: Folders — authorization on per-folder endpoints
**Environment**: dev (https://dev-api.doable.me)

## Steps to Reproduce

1. Authenticate as user A (member of workspace A only)
2. Obtain a folder ID from workspace B (e.g., via a second test account)
3. Any of:
   - GET /folders/{folderFromWorkspaceB}
   - PATCH /folders/{folderFromWorkspaceB} with `{"name": "hacked"}`
   - DELETE /folders/{folderFromWorkspaceB}

## Expected

403 Forbidden — user A is not a member of workspace B

## Actual

- GET: 200 OK — folder data returned across workspace boundary
- PATCH: 200 OK — folder modified across workspace boundary
- DELETE: 204 No Content — folder deleted across workspace boundary

## Impact

Critical authorization gap. Any authenticated user can read, modify, or delete folders in any workspace if they know (or guess) the folder UUID. This is an IDOR (Insecure Direct Object Reference) vulnerability.

TC-FOLDER-GET-002, TC-FOLDER-UPDATE-AUTH, TC-FOLDER-DELETE-AUTH
