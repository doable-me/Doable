# BUG-014: SSE `done` event has empty data — missing messageId and creditsUsed

**Date:** 2026-05-14
**Suite:** 05-ai-chat
**Severity:** High
**Test Cases:** TC-AI-CHAT-SEND-017

## Description
The spec requires the SSE `done` event to contain `messageId`, `creditsUsed:1`, and `tokenUsage` totals. The actual `done` event payload is an empty object `{}`.

## Reproduction
```
POST https://dev-api.doable.me/projects/:id/chat
{"content":"hello","mode":"agent"}
```

## Actual SSE Events
```
data: {"type":"done","data":{}}
data: {"type":"usage","data":{"promptTokens":17614,"completionTokens":35,...,"model":"MiniMax-M2.7-highspeed","tokensAvailable":true}}
```

## Expected
```
data: {"type":"done","data":{"messageId":"<uuid>","creditsUsed":1,"tokenUsage":{"prompt":...,"completion":...}}}
```

## Impact
Clients cannot determine the persisted message ID from the stream or confirm credit deduction. Token usage IS available in a separate `usage` event but the `done` event lacks the required fields.
