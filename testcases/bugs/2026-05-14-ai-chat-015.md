# BUG-015: Emoji/multi-byte Unicode content stored as "??????" — UTF-8 encoding corruption

**Date:** 2026-05-14
**Suite:** 05-ai-chat
**Severity:** Critical
**Test Cases:** TC-AI-CHAT-SEND-035

## Description
Multi-byte Unicode emoji characters sent in chat content are corrupted in storage. The characters are replaced with "?" (question marks), indicating a UTF-8 to latin1 encoding mismatch between the API and the database.

## Reproduction
```
POST https://dev-api.doable.me/projects/:id/chat
{"content":"Say: 🎉🚀💡 — test emoji","mode":"agent"}
```
Then:
```
GET https://dev-api.doable.me/projects/:id/chat/history
```

## Actual Stored Content
```
'Say: ?????? \xe2\x80\x94 test emoji'
```
(Emoji characters replaced with ?, em-dash stored as raw bytes)

## Expected
Content stored byte-identical: `"Say: 🎉🚀💡 — test emoji"`

## Impact
Any user message containing emoji, CJK characters, or other non-ASCII Unicode is silently corrupted in the database. This is a data integrity issue affecting all users globally. The AI responds to the corrupted content, not the original.
