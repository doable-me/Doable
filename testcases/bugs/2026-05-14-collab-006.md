# BUG-COLLAB-006: /github/status returns 404

**Date**: 2026-05-14
**Environment**: dev (dev-api.doable.me)
**Severity**: LOW
**Area**: API / GitHub Integration

## Summary
The GitHub status endpoint `GET /{projectId}/github/status` returns HTTP 404, indicating this endpoint is not implemented or not registered.

## Steps to Reproduce
1. Open a project in the editor
2. Observe browser console

## Expected
`GET /{projectId}/github/status` → 200 with GitHub connection status

## Actual
`GET https://dev-api.doable.me/f42fcc90-7789-465f-853c-d05ef54c5915/github/status` → 404

## Evidence
Browser console:
- `Failed to load resource: 404 () @ https://dev-api.doable.me/f42fcc90-7789-465f-853c-d05ef54c5915/github/status`

## Note
URL pattern `/{projectId}/github/status` (project ID at root level rather than `/projects/{id}/github/status`) may indicate a routing misconfiguration.

## Impact
GitHub integration status indicator in editor is always broken/missing.
