# BUG-API-016: GET /workspaces/:wid/ai-settings returns 404

**Date:** 2026-05-14
**Severity:** high
**Area:** TC-API-PROV-021, TC-API-PROV-045

## Summary
`GET /workspaces/:wid/ai-settings` returns 404. Also `GET /workspaces/:wid/ai-settings/copilot` returns 404. These endpoints are documented in routes.ts and test specs.

## Steps to Reproduce
```bash
curl "https://dev-api.doable.me/workspaces/<wid>/ai-settings" \
  -H "Authorization: Bearer $TOKEN"
# {"error":"Not Found","path":"/workspaces/<wid>/ai-settings"}

curl "https://dev-api.doable.me/workspaces/<wid>/ai-settings/copilot" \
  -H "Authorization: Bearer $TOKEN"
# {"error":"Not Found"}
```

## Expected
200 with AI settings. Note: `GET /workspaces/:wid/ai-settings/providers` DOES work (200).

## Impact
Workspace-level AI settings (default model, copilot overrides) cannot be read or configured via API.
