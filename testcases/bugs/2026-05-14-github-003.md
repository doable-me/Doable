# BUG-GH-003: GitHub API routes return 404 (not registered)

**Date**: 2026-05-14
**Severity**: High
**Feature**: GitHub — API surface (commits, push, pull, import, connect)
**Environment**: dev (https://dev-api.doable.me)

## Affected Endpoints

| Endpoint | Method | Expected | Actual |
|---|---|---|---|
| /github/connect | GET | 200/redirect | 404 |
| /projects/:id/github/commits | GET | 200 list | 404 |
| /projects/:id/github/push | POST | 200/202 | 404 |
| /projects/:id/github/pull | POST | 200/202 | 404 |
| /github/import | POST | 202 | 404 |

## Steps to Reproduce

1. Authenticate as owner-pro@doable.me
2. Call any of the above endpoints with a valid project ID

## Expected

Endpoints respond with appropriate status (200/201/202)

## Actual

404 Not Found — routes are not registered in the API server

## Impact

All GitHub sync functionality (push, pull, import, commit history) is non-functional. The GitHub integration exists only as a UI shell with no backend.

## Test Cases

TC-GH-CONNECT-API-001, TC-GH-COMMITS-001, TC-GH-PUSH-001, TC-GH-PULL-001, TC-GH-IMPORT-001
