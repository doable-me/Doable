# BUG-FOLDER-006: PATCH /projects/:id with folderId returns 403 for project owner

**Date**: 2026-05-14
**Severity**: High
**Feature**: Folders — moving projects into folders
**Environment**: dev (https://dev-api.doable.me)

## Steps to Reproduce

1. Authenticate as owner-pro@doable.me (project owner, role=owner)
2. Create a folder in the workspace
3. PATCH /projects/{projectId} with body `{"folderId": "{folderId}"}`

## Expected

200 OK — project moved into the folder; folderId updated in response

## Actual

403 Forbidden — response body: `{"error": "Viewers cannot edit projects"}`

## Impact

Project owners cannot move their own projects into folders. The role check is incorrectly applied — the user is an owner, not a viewer. This blocks the entire folder organization feature for projects.

## Notes

The 403 message "Viewers cannot edit projects" suggests the role-resolution middleware is returning "viewer" for a user who should be "owner". Possible cause: the project PATCH endpoint uses a different role-lookup path than other project endpoints.
