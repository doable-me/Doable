# BUG-ADMIN-009: Web /admin page returns 200 HTML to unauthenticated users — no redirect to login

**Date:** 2026-05-14
**Severity:** High / P0
**Test Cases:** TC-ADMIN-DASHBOARD-002, TC-ADMIN-DASHBOARD-003, TC-ADMIN-DASHBOARD-019

## Summary
GET https://dev.doable.me/admin (unauthenticated, no cookies) returns HTTP 200 with admin HTML content instead of redirecting to /login. The page body contains admin-related content ("admin", "dashboard", "projects") visible without authentication.

## Steps to Reproduce
```
curl -s https://dev.doable.me/admin
```

## Expected
HTTP 302/307 redirect to /login?next=/admin. No admin content visible.

## Actual
HTTP 200 with full Next.js HTML page containing admin-related markup.

Body snippet shows Next.js shell with admin page content (SSR or client-side shell). The auth check appears to be client-side only — the server serves the admin shell to anyone.

## Note
The API endpoints correctly return 401/403. The vulnerability is in the Next.js frontend serving the admin page shell to unauthenticated users. Sensitive data may be exposed if any admin data is SSR-rendered into the HTML.

## Impact
Admin UI shell exposed to unauthenticated users. If any data is server-side rendered into the HTML, it would be visible without login. Client-side-only auth guard can be bypassed by disabling JavaScript.
