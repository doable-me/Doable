### Patterns learned
- services/api/src/routes.ts is the canonical mount table (88 source files, 871+ route registrations, but many paths are nested under /workspaces/:id/ or /projects/:id/).
- Hono middleware is positional: `billingRoutes.use("/*", authMiddleware)` only affects routes registered AFTER it. The matrix v5 update for /billing/topup/packages exposed this neatly.
- Cloudflare Tunnel + Hono trailing-slash 308 redirects strip `Authorization` headers in Node fetch (BUG-R10-TRAILING-SLASH-AUTH-DROP-001) — real-client footgun.
- The "reuse existing .env" branch of setup-server.sh is the silent hole — fresh installs are tested often, .env-reuse paths much less so. Worth a future audit pass for other vars.
- Rate-limit collisions (registerRateLimiter 5/hour/IP, loginRateLimiter, forgotPasswordRateLimiter 3/hour) make multi-role seeding slow; matrix gracefully handles via SKIPPED-NO-TOKEN.

# Ralph round 17 — /admin "A.map is not a function" P0 fix

## Mission
User reports /admin on dev.doable.me is broken with "Something went wrong" and "A.map is not a function".

## Reproduction (claude-in-chrome on dev)
- Logged in as admin2@doable.me (isPlatformAdmin: true)
- Navigated to https://dev.doable.me/admin
- Captured: TypeError: A.map is not a function — chunk 02_qi-vb43azw.js:1:198035
- /admin/features 200 (array OK), /admin/frameworks 200 ({frameworks,defaultFramework} OK), /admin/users 200 ({data,total,limit,offset} - BREAKS HOOK)

## Root cause (BUG-ADMIN-012, regression of BUG-ADMIN-005)
Commit 05b622dc enriched GET /admin/users with plan/AI/credits fields but
also flipped the response shape from a flat snake_case array to a
{ data, total, limit, offset } envelope with camelCase keys. The single
consumer use-platform-admin.ts does setUsers(data) then admin/page.tsx
does users.map(...), which throws because users is now an object.

## Fix
1. services/api/src/routes/admin-users.ts — GET /users returns a flat
   snake_case array (matching every consumer in admin-components.tsx,
   user-management-panel.tsx, page.tsx, and the AdminUser type).
   Enriched fields (plan, ai_source, model, daily/monthly/rollover
   credits, workspace_id) retained in snake_case.
2. apps/web/src/hooks/use-platform-admin.ts — loadFeatures + loadUsers
   now defensively unwrap { data } envelopes and Array.isArray-coerce
   before setState. Future drift can't reproduce this crash class.
3. scripts/r17-test-admin-users-contract.ts — 26-assertion contract probe.
   Run with `pnpm exec tsx scripts/r17-test-admin-users-contract.ts`.
   Pins flat-array shape, snake_case keys, no camelCase variants.
4. testcases/bugs/2026-05-15-admin-012.md — bug ticket with full repro,
   RCA, and prevention notes.
