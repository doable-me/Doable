## What works end-to-end after Wave 4

1. **Vite-React projects:** unchanged behavior. Every hardcoded constant
   now flows through `viteReactAdapter.defaults`.
2. **Next.js projects:** scaffold from template → project row gets
   framework_id="nextjs-app" → dev-server spawns `next dev` → builder
   runs `next build` → AI receives Next.js system prompt + 4 nextjs-app
   skill files (server-actions, route-handlers, connector-fetch,
   env-conventions) → publish triggers nodeStandaloneAdapter.start()
   which writes the systemd drop-in + addProcessRoute() in Caddy +
   project_runtime row inserted.
3. **Connector bridge for static SPAs (PRD 10) — fully wired:**
   - Editor opens project → preview iframe loads.
   - Iframe's CONNECTOR_BRIDGE_SNIPPET (injected by proxy-handler)
     posts "doable:connector-proxy-ready" to parent.
   - Editor's useEffect fetches a 15-min JWT via
     `POST /projects/:id/connector-proxy-token` (user-session-authed).
   - Editor postMessages token back; in-iframe helper resumes pending
     callConnector() promises.
   - Generated SPA can now call `window.__doable.callConnector(integration, action, props)`
     and the proxy validates JWT, checks `.doable/connector-allowlist.json`,
     runs the Activepieces action, audits the call.
4. **Per-OS sandbox auto-selects:** Psroot (Win+psroot.exe), systemd
   or bubblewrap (Linux), sandbox-exec (macOS).
5. **Runtime control routes live:** GET /projects/:id/runtime,
   POST .../restart, GET .../logs.
6. **dovault config-guard per-framework:** vite-react, nextjs-app,
   nuxt, sveltekit, astro each have their own locked-config templates.
