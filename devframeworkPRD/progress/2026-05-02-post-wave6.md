## What ships in Wave 5 + Wave 6

### Phase 5 — Per-app supervision + runtime control
- `services/api/src/runtime/supervisor.ts` — subscribes to `journalctl
  -u 'doable-app@*.service' -f -o json --no-pager`. Parses unit state
  changes (Started/Reloaded → running, Failed → failed + bump
  fail_count, Stopped/Deactivated → stopped) into `project_runtime`.
  Auto-restarts journalctl after 5s if it dies. Reconciles on boot via
  systemctl is-active.
- `services/api/src/index.ts` calls `startSupervisor()` after framework
  init.
- No-op on Windows / macOS dev — logs warning + returns dummy handle.

### PRD 03 — Build event protocol
- `services/api/src/build-events/types.ts` — full event vocabulary
  per PRD §2 (BuildLog raw + 8 structured event types).
- `services/api/src/build-events/buffer.ts` — per-project ring buffer
  (capacity 5000, overrideable via BUILD_BUFFER_SIZE). Implements
  PRD §6.3 drop policy: structured events never dropped; build_log
  evicted first; on hard overflow capacity doubles once to 50_000
  then drops with warn.
- `services/api/src/build-events/publisher.ts` — `BuildEventPublisher`
  with per-stream batching (50 lines / 8KiB / 50ms whichever first).
  attach(child, buildId) hooks stdout+stderr through the filter chain
  and into the buffer. publish() for direct structured events.
  subscribe(projectId, fn) returns unsubscribe.
- `services/api/src/routes/build-stream.ts` — `GET
  /projects/:id/build/stream` SSE endpoint with cursor protocol
  (latest|start|N), 25s keep-alive, replay-then-tail semantics.
- `services/api/src/projects/dev-server-start.ts` — wires publisher to
  the dev-server child. envSecrets sourced from resolved userEnvVars
  for env-values redaction.

### PRD 04 — Redaction filter chain
- `services/api/src/build-events/filters/{types,chain}.ts` — fail-
  closed LogFilterChain.
- 6 always-on filters: env-values (literal-match), shapes (AWS, GitHub,
  Supabase, Slack, Stripe, Anthropic, OpenAI, JWT), entropy (Shannon
  > 4.0 bits/char on words length>=32), paths (host-path leak), urls
  (userinfo + query strip), usernames (literal OS user).
- `buildDefaultFilters()` returns the canonical chain in PRD §2.1
  order.

### Templates
- `nuxt-blank.ts`, `sveltekit-blank.ts`, `astro-blank.ts` registered.
  Users can now scaffold any of the 5 supported frameworks from the
  template picker (vite-react, nextjs, nuxt, sveltekit, astro).

## End-to-end paths now working

1. **Vite-React projects** — unchanged.
2. **Next.js projects** — scaffold → adapter resolves → next dev →
   AI gets Next.js prompt + 4 nextjs-app skill files → publish →
   nodeStandaloneAdapter writes systemd drop-in → addProcessRoute in
   Caddy → project_runtime row inserted → supervisor watches state.
3. **Static SPA + connector bridge (PRD 10)** — preview iframe loads →
   announces ready → editor fetches JWT → postMessages back → SPA
   calls window.__doable.callConnector() → proxy validates JWT +
   allowlist + runs Activepieces action + audits.
4. **Build progress visibility (NEW)** — dev-server stdout flows
   through filter chain → ring buffer → editor (or any client) tails
   `GET /projects/:id/build/stream` and receives raw build_log
   events as they happen, with credentials redacted. PRD 05 React
   surface to render this is the next major UX win, but the data
   stream is live.
